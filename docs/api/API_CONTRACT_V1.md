# Student Evaluation System — API Contract v1

## Overview

This document defines the API contract for the Student Evaluation System. All APIs follow RESTful conventions and return JSON responses.

### Base URL

``` plain
/api/v1/
```

### Authentication

- JWT-based authentication using SimpleJWT
- Access token stored in HTTP-only cookie (`access_token`)
- Refresh token stored in HTTP-only cookie (`refresh_token`)
- Login endpoint: `POST /api/v1/users/auth/login/`
- Token refresh: `POST /api/v1/users/auth/refresh/`

### Response Envelope

All responses follow this envelope structure:

**Success (200/201):**

```json
{
  "ok": true,
  "data": { ... },
  "meta": {
    "request_id": "req_abc123",
    "timestamp": "2026-04-09T12:00:00Z"
  }
}
```

**Error (4xx/5xx):**

```json
{
  "ok": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Invalid request payload",
    "details": [...],
    "retryable": false
  },
  "meta": {
    "request_id": "req_abc123",
    "timestamp": "2026-04-09T12:00:00Z"
  }
}
```

### Timestamps

- All timestamps in ISO-8601 UTC format (e.g., `2026-04-09T12:00:00Z`)
- Generated by server

## Authentication Endpoints

### POST /users/auth/login/

Login and receive JWT tokens.

**Request:**

```json
{
  "username": "student123",
  "password": "securepassword"
}
```

**Response (200):**

```json
{
  "ok": true,
  "data": {
    "user": {
      "id": 1,
      "username": "student123",
      "email": "student@example.edu",
      "first_name": "John",
      "last_name": "Doe",
      "role": "student",
      "department": "Computer Science",
      "university": "State University",
      "is_active": true,
      "is_staff": false,
      "date_joined": "2025-09-01T00:00:00Z"
    }
  },
  "meta": {
    "request_id": "req_001",
    "timestamp": "2026-04-09T12:00:00Z"
  }
}
```

**Errors:**

- `400` - Missing username or password
- `401` - Invalid credentials
- `403` - User account disabled
- `429` - Rate limited (5 attempts/minute)

---

Refresh access token using refresh token cookie.

**Response (200):**

```json
{
  "ok": true,
  "data": {
    "access": "eyJ0eXAiOiJKV1Q...",
    "refresh": "eyJ0eXAiOiJKV1Q..."
  }
}
```

**Response (200):**

**Errors:**

- `401` - Current password incorrect

## Status Codes

| Code | Meaning |
| ------ | --------- |
| 200 | OK |
| 201 | Created |
| 400 | Bad Request - Validation error |
| 401 | Unauthorized - Not authenticated |
| 403 | Forbidden - Not authorized |
| 404 | Not Found |
| 409 | Conflict - Duplicate or constraint violation |
| 429 | Rate Limited |
| 500 | Internal Server Error |

## Error Response Schema

```json
{
  "ok": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Human readable message",
    "details": [
      { "field": "field_name", "reason": "reason_code" }
    ],
    "retryable": false
  },
  "meta": {
    "request_id": "req_abc123",
    "timestamp": "2026-04-09T12:00:00Z"
  }
}
```

## Rate Limits

- Login: 5 requests/minute per IP
- File uploads: 10 requests/minute
- General API: 60 requests/minute per user

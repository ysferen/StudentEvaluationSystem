# Local Development & Implementation Guide

## Prerequisites
- Python 3.12+ and `uv` installed
- Node.js 18+ installed
- PostgreSQL (if using locally via Docker, otherwise defaults to SQLite)

## Local Development Workflow

### Backend (uv preferred)
To set up and run the Django application:
```bash
cd backend
uv sync
cd student_evaluation_system
uv run python manage.py migrate
uv run python manage.py runserver
```

**Lint & Test:**
```bash
uv run ruff check .
uv run pytest
```

### Frontend
To set up and run the React/Vite application:
```bash
cd frontend
npm install
npm run dev
```

**Lint & Test:**
```bash
npm run lint
npm run test -- --run
```

## Environment Variables
*(Copy the respective `.env.example` files to `.env` and populate the values)*

- **Backend (`backend/.env`):** `SECRET_KEY`, `DEBUG`, `ALLOWED_HOSTS`, `DATABASE_URL`, `CORS_ALLOWED_ORIGINS`, `ACCESS_TOKEN_LIFETIME_MINUTES`, `REFRESH_TOKEN_LIFETIME_DAYS`, `THROTTLE_RATES`, `MAX_UPLOAD_SIZE_MB`, `LOG_LEVEL`
- **Frontend (`frontend/.env.development` & `.env.production`):** `VITE_API_URL`, `VITE_API_BASE_PATH`, `VITE_ENABLE_DEBUG`, `VITE_ENABLE_MOCK_API`

## API Client Generation
Frontend API clients are generated by Orval and **should not be hand-edited**.
The source of truth is the backend OpenAPI schema (`backend/student_evaluation_system/schema.yml`).

To regenerate the frontend clients after a backend change:
```bash
cd frontend
npm run generate:api
```

## Testing and Quality Gates
- **Backend:** `pytest` + `pytest-django`.
  - Coverage threshold: **70%**
  - Ruff complexity limit: **10**
  - Ruff line length: **127**
- **Frontend:** Vitest (jsdom).
  - ESLint + TS strict mode enabled.

## Common Implementation Patterns

**Backend Feature Workflow:**
1. Update model/service/serializer/view.
2. Register the route.
3. Run `makemigrations` and `migrate`.
4. Run tests (`uv run pytest`).
5. Regenerate backend API schema (`uv run python student_evaluation_system/manage.py spectacular --file schema.yml`).

**Frontend Feature Workflow:**
1. Implement the feature under `features/<feature>/`.
2. Wire the route in `App.tsx`.
3. Use generated API clients (do not write manual fetch calls).
4. Run tests and linter.

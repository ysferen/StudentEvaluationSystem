# Local Development & Implementation Guide

## Local Development Workflow

### Backend (uv preferred)
cd backend
uv sync
cd student_evaluation_system
uv run python manage.py migrate
uv run python manage.py runserver
Lint/Test: uv run ruff check ., uv run pytest

### Frontend

cd frontend
npm install
npm run dev
Lint/Test: npm run lint, npm run test -- --run

### Environment Variables
Backend (.env): SECRET_KEY, DEBUG, ALLOWED_HOSTS, DATABASE_URL, CORS_ALLOWED_ORIGINS, ACCESS_TOKEN_LIFETIME_MINUTES, REFRESH_TOKEN_LIFETIME_DAYS, THROTTLE_RATES, MAX_UPLOAD_SIZE_MB, LOG_LEVEL

Frontend (.env.development/.production): VITE_API_URL, VITE_API_BASE_PATH, VITE_ENABLE_DEBUG, VITE_ENABLE_MOCK_API

### API Client Generation
Clients are generated by Orval and should not be hand-edited.
Source: backend/student_evaluation_system/schema.yml

### Bash
cd frontend
npm run generate:api
Testing and Quality Gates
Backend: pytest + pytest-django. 70% coverage threshold. Ruff complexity limit: 10. Ruff line length: 127.

Frontend: Vitest (jsdom). ESLint + TS strict mode.

### Common Implementation Patterns
Backend Feature: Update model/service/serializer/view -> Register route -> Make migrations -> Run tests -> Regenerate API schema.

Frontend Feature: Implement under features/<feature>/ -> Wire route in App.tsx -> Use generated API clients -> Run tests.
